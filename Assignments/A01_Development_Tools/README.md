# A01 – Development Tools for Edge AI and IoT

## Overview

This assignment focused on understanding the main tools used in Edge AI and IoT development. The goal was to research and explain how different platforms support coding, model building, deployment, collaboration, and device-level AI development.

The tools covered in this assignment were:

- Visual Studio Code
- Node.js
- Edge Impulse CLI
- TensorFlow and TensorFlow Lite
- Google Colab
- Generative AI coding tools

## Problem / Purpose

Edge AI and IoT projects usually require more than one tool. A developer may need an editor for writing code, a framework for training models, a way to deploy models to devices, and tools for collaboration or debugging.

The purpose of this assignment was to understand how these tools fit together in a real Edge AI workflow.

## Approach

I researched each tool by focusing on three main areas:

1. **Description** – what the tool is  
2. **Purpose** – why it is used  
3. **Typical use cases** – how it applies to Edge AI or IoT development  

This helped me compare the tools not just by definition, but by how they would actually be used in a project.

## What I Learned

I learned that Edge AI development is not just about building a machine learning model. It also involves choosing the right tools for coding, testing, converting, deploying, and managing the system.

Some important takeaways:

- **VS Code** is useful because it combines coding, debugging, Git support, and extensions in one place.
- **Node.js** can support IoT systems by handling APIs, real-time communication, and sensor data.
- **Edge Impulse CLI** helps connect sensor data collection, model training, and deployment for edge devices.
- **TensorFlow** is mainly used for training models, while **TensorFlow Lite** is used to deploy models efficiently on mobile or embedded devices.
- **Google Colab** is helpful for quickly testing machine learning code without needing a powerful local computer.
- **Generative AI coding tools** can help with code generation, debugging, and learning unfamiliar libraries.

## Challenges

One challenge was understanding how all the tools connect to each other. At first, it was easy to see them as separate tools, but the assignment helped me understand that they can be part of one complete workflow.

For example, a developer might use Google Colab to train a model, TensorFlow Lite to convert it, VS Code to write deployment code, and Edge Impulse CLI to test it on an edge device.

## Results / Outcome

By the end of the assignment, I had a clearer understanding of the development ecosystem used in Edge AI and IoT. This assignment gave me a foundation for later labs where I worked more directly with TensorFlow Lite, forecasting, and IIoT system design.
