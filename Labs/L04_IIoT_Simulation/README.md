# L04 – IIoT Simulation

## Overview

The goal of this lab was to explore how Industrial Internet of Things (IIoT) systems can be simulated to better understand network behavior and system interactions.

Instead of building everything from scratch, this lab used an existing simulation project to observe how devices communicate, how data flows through the system, and how different parameters can affect performance.

---

## What Was Done

- Worked with an existing IIoT simulation project
- Explored how devices communicate within a simulated network
- Observed how different system parameters impact performance
- Analyzed the structure and behavior of an IIoT system

Simulation Repository:
https://github.com/ethanphaan/iiot_simulation

---

## Lessons to be learned

This lab can help one understand that IIoT systems are more than just individual devices. They are complex systems where many components interact at the same time.

Some key takeaways:

- IIoT systems involve multiple devices communicating simultaneously  
- Network conditions (like traffic and collisions) can affect performance  
- Simulation is useful for testing systems before real deployment  
- Small changes in parameters can significantly impact system behavior  

---

## Challenges

- Understanding how the simulation was structured
- Interpreting results without building the system from scratch
- Connecting simulation behavior to real-world IIoT systems

---

## Real-World Relevance

In real IIoT environments, testing directly on physical systems can be expensive and risky.

Simulations allow engineers to:
- Test system performance  
- Identify bottlenecks  
- Optimize configurations  

before deploying systems in real environments.

---

## Conclusion

This lab introduced me to the importance of simulation in IIoT development. It showed how complex systems can be analyzed and improved before real-world deployment, which is a critical step in building reliable and efficient systems.
