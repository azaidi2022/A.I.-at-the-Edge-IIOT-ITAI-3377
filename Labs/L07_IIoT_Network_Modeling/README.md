# L07 – IIoT Network Modeling

## Overview

The purpose of this lab was to analyze how different factors affect the performance of an Industrial Internet of Things (IIoT) network.

The focus was on understanding how network conditions influence key performance metrics, particularly how systems behave when multiple devices are communicating at the same time.

This lab was more focused on modeling and experimentation rather than just implementing a standard machine learning model.

---

## What I Did

- Explored IIoT network behavior using a dataset
- Analyzed how different parameters affect system performance
- Applied data preprocessing and feature engineering
- Built models to better understand relationships in the data
- Evaluated how network conditions impact results

---

## What I Learned

This lab helped me understand that IIoT systems involve trade-offs between different performance metrics.

Some key takeaways:

- Network performance depends on multiple interacting factors  
- Increasing activity in a network can improve some metrics but worsen others  
- Feature engineering is important for capturing real system behavior  
- Modeling real-world systems requires understanding both data and system dynamics  

---

## Challenges

- Understanding how different network variables interact  
- Interpreting results beyond just model outputs  
- Representing real-world system behavior using data  

---

## Key Insight

One of the biggest takeaways from this lab is that **system-level understanding is just as important as model performance**.

Instead of only focusing on prediction accuracy, it is important to understand how and why the system behaves the way it does.

---

## Real-World Relevance

IIoT networks are used in environments like:

- Smart factories  
- Industrial monitoring systems  
- Healthcare systems  

In these environments, understanding how network conditions affect performance is critical for reliability and efficiency.
